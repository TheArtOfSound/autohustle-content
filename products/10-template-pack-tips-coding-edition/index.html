Template 7: [API Specification Template – OpenAPI Draft]

When to use: When defining a new REST API endpoint for team review

```

API Specification: [Endpoint Name]

Version: [e.g., v1.0]

Author: [Your Name]

Last Updated: [YYYY-MM-DD]

Endpoint

[HTTP METHOD] /api/[version]/[resource-path]

Summary: [e.g., "Retrieve user profile data"]

Request

Headers

  • Authorization: Bearer [token]
  • Content-Type: application/json

Path Parameters

| Name | Type | Required | Description |

|------|------|----------|-------------|

| [userId] | string | yes | Unique ID of the user |

Query Parameters

| Name | Type | Required | Description |

|------|------|----------|-------------|

| [fields] | string | no | Comma-separated list of fields to return |

Body (if applicable)

```json

{

"[field]": "[value]",

"[nested]": {

"[subfield]": "[value]"

}

}

```

Response

200 OK

```json

{

"status": "success",

"data": {

"id": "[userId]",

"name": "[User Name]",

"email": "[User Email]"

}

}

```

400 Bad Request

```json

{

"status": "error",

"code": "invalid_input",

"message": "[Description]"

}

```

404 Not Found

[Describe when this occurs]

Authentication

[OAuth2 / API Key / JWT – specify method]

Rate Limiting

[Requests per minute: e.g., 100]

Notes

  • [e.g., "Caches response for 5 minutes"]
  • [e.g., "Logs all access for audit"]

```
