Lesson 1: Manipulating Files using Python's Built-in Functions

Python provides several built-in functions for manipulating files. In this lesson, we'll cover the most commonly used ones:

  • `open()`: Opens a file in read or write mode.
  • `read()`: Reads the contents of a file.
  • `write()`: Writes data to a file.
  • `close()`: Closes a file.

Here's an example code snippet that demonstrates how to use these functions:

```python

Open a file in write mode

file = open("example.txt", "w")

file.write("Hello, World!")

file.close()

Read the contents of a file

with open("example.txt", "r") as file:

print(file.read())

Write data to a file

data = "This is some data"

with open("example.txt", "a") as file:

file.write("\n" + data)

Close a file (not necessary with 'with' statement)

file = open("example.txt", "r")

print(file.read())

file.close()

```

Lesson 3: Understanding File Permissions and Security

File permissions control access to files and folders on your system. In this lesson, we'll cover the basics of file permissions:

  • Read (r): Allows reading the contents of a file.
  • Write (w): Allows writing data to a file.
  • Execute (x): Allows running a script or program from within a folder.

Here's an example code snippet that demonstrates how to use Python's `os` module to change file permissions:

```python

import os

Get the current permissions of a file

file = "example.txt"

permissions = os.stat(file).st_mode

print(permissions)

Change the read permission of a file

os.chmod(file, permissions | 0o400)

Change the write permission of a folder

folder = "/path/to/folder"

os.chmod(folder, permissions | 0o200)

```

Lesson 1: Using Exceptions and Error Handling in Scripts

In any automation script, errors are inevitable. They can occur due to various reasons such as file not found, network connectivity issues, or invalid data. Effective error handling is crucial to ensure that your scripts remain stable and can recover from failures.

#### What are exceptions?

Exceptions are events that occur during the execution of a program. In Python, exceptions are handled using the `try-except` block.

```python

try:

Code that might raise an exception

except Exception as e:

Handle the exception

```

#### Example: Handling File Not Found

Suppose we have a script that tries to read a file from a specific location. If the file does not exist, it will raise a `FileNotFoundError`.

```python

import os

def read_file(file_path):

try:

with open(file_path, 'r') as f:

return f.read()

except FileNotFoundError:

print(f"File {file_path} not found.")

read_file('non_existent_file.txt')

```

In this example, we catch the `FileNotFoundError` exception and print a friendly error message.

Lesson 2: Applying Object-Oriented Programming Concepts

Object-oriented programming (OOP) is a fundamental concept in software development. In Python, we can leverage OOP principles to write more modular and reusable code.

#### What are classes?

Classes are templates for creating objects. An object represents an entity with properties and behavior.

```python

class Car:

def __init__(self, color, model):

def start_engine(self):
```

#### Example: Creating a reusable logger class

Suppose we have multiple scripts that need to log events. Instead of duplicating the logging code, we can create a reusable `Logger` class.

```python

import datetime

class Logger:

def __init__(self):

self.log_file = 'log.txt'

def log(self, message):

with open(self.log_file, 'a') as f:

f.write(f"{datetime.datetime.now()} - {message}\n")

logger = Logger()

logger.log('Script started.')

```

In this example, we create a `Logger` class that has a single method `log()`. We can reuse this class across multiple scripts to log events.

Lesson 3: Creating Reusable Code Modules

Modules are self-contained pieces of code that provide a specific functionality. In Python, we use the `import` statement to import modules.

```python

```

#### Example: Creating a reusable data loader module

Suppose we have multiple scripts that need to load data from a CSV file. Instead of duplicating the loading code, we can create a reusable `DataLoader` class.

```python

import csv

class DataLoader:

def __init__(self, file_path):

self.file_path = file_path

def load_data(self):

with open(self.file_path, 'r') as f:

reader = csv.reader(f)

data = list(reader)

return data

data_loader = DataLoader('data.csv')

data = data_loader.load_data()

```

In this example, we create a `DataLoader` class that has a single method `load_data()`. We can reuse this class across multiple scripts to load data from CSV files.
