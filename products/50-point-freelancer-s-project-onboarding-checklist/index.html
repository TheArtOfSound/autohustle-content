50-Point Freelancer's Project Onboarding Checklist

Updated: March 04, 2026

50-point checklist to ensure all project details are covered before starting work.

What's Inside

  • 50 comprehensive checklist items
  • organized by project phase
  • easy to follow format
  • [ ] Item 1: Confirm the client’s full name and preferred contact method (email, phone, messaging app)
  • [ ] Item 2: Request the client’s business name and official company registration details if applicable
  • [ ] Item 3: Schedule a kickoff call to discuss project goals and expectations
  • [ ] Item 4: Send a calendar invite for the kickoff meeting with agenda attached
  • [ ] Item 5: Prepare and share a list of discovery questions before the kickoff call
  • [ ] Item 6: Identify the client’s primary decision-maker and backup contact
  • [ ] Item 7: Clarify the client’s preferred communication frequency and response time expectations
  • [ ] Item 8: Ask the client to describe the project’s primary goal in one sentence
  • [ ] Item 9: Define success metrics or KPIs the client will use to evaluate the project
  • [ ] Item 10: Request access to relevant brand guidelines (logo, colors, fonts, tone of voice)
  • [ ] Item 11: Ask for examples of work they admire or want to emulate
  • [ ] Item 12: Confirm whether they have existing content, materials, or assets to share
  • [ ] Item 13: Request login credentials or access to necessary platforms (e.g., website CMS, social media)
  • [ ] Item 14: Set up a secure method for sharing sensitive information (e.g., password manager, encrypted email)
  • [ ] Item 15: Create a shared project folder (e.g., Google Drive, Dropbox) for documents and assets
  • [ ] Item 16: Establish a central communication channel (e.g., Slack, email thread, project tool)
  • [ ] Item 17: Choose a project management tool (e.g., Trello, Asana, ClickUp) and invite the client if needed
  • [ ] Item 18: Define project phases or milestones with clear deliverables
  • [ ] Item 19: Draft a detailed scope of work document outlining all included tasks
  • [ ] Item 20: Explicitly list out-of-scope items to prevent scope creep
  • [ ] Item 21: Agree on the number of revision rounds included per deliverable
  • [ ] Item 22: Set hard deadlines for each project milestone
  • [ ] Item 23: Confirm the project start and end dates in writing
  • [ ] Item 24: Break down the timeline into weekly action items
  • [ ] Item 25: Identify any third-party dependencies or approvals that could cause delays
  • [ ] Item 26: Determine the client’s availability during key project phases
  • [ ] Item 27: Agree on file formats for final deliverables (e.g., PDF, .PSD, .MP4)
  • [ ] Item 28: Specify naming conventions and organization standards for shared files
  • [ ] Item 29: Confirm the total project fee and payment structure (e.g., 50% upfront, 50% on completion)
  • [ ] Item 30: Issue a formal invoice for the initial deposit with payment due date
  • [ ] Item 31: Agree on acceptable payment methods (e.g., bank transfer, PayPal, Stripe)
  • [ ] Item 32: Outline late payment penalties or fees in the contract
  • [ ] Item 33: Draft a written contract covering scope, timeline, payment, and revisions
  • [ ] Item 34: Include a clause for additional fees if scope changes occur
  • [ ] Item 35: Add a termination clause outlining conditions for ending the project early
  • [ ] Item 36: Specify intellectual property rights and ownership of final deliverables
  • [ ] Item 37: Require both parties to sign the contract before work begins
  • [ ] Item 38: Store a signed copy of the contract in the shared project folder
  • [ ] Item 39: Request the signed contract and deposit payment before starting Phase 1
  • [ ] Item 40: Confirm tax requirements and whether a W-9 or VAT number is needed
  • [ ] Item 41: Set up automated payment reminders for upcoming installments
  • [ ] Item 42: Define the process for client feedback and approval on deliverables
  • [ ] Item 43: Establish a turnaround time for client feedback to keep the project on track
  • [ ] Item 44: Create a content or asset submission deadline for the client if their input is required
  • [ ] Item 45: Document any technical or platform-specific requirements for the deliverables
  • [ ] Item 46: Perform a test run of any tools or integrations needed for the project
  • [ ] Item 47: Confirm the client’s preferred method for receiving progress updates (e.g., weekly email)
  • [ ] Item 48: Schedule regular check-in meetings at consistent intervals
  • [ ] Item 49: Send a written onboarding summary confirming all agreed-upon details
  • [ ] Item 50: Officially mark the project as “In Progress” in your workload tracker
